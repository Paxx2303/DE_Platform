**1. Nội dung công việc được giao**



Nhận mô tả công việc và định hướng nghiên cứu cho bài toán xây dựng LLM Chatbot/Data Agent có khả năng truy xuất chính xác thông tin từ các datasource dựa trên câu hỏi tự nhiên của người dùng.

Sản phẩm hướng đến là hệ thống có thể chuyển đổi Natural Language thành truy vấn phù hợp như SQL, DAX hoặc các dạng query khác, phục vụ việc khai thác dữ liệu từ Data Platform.



**2. Trách nhiệm chính**



Nghiên cứu, đánh giá và đề xuất hướng thiết kế Data Model/Data Foundation phù hợp để AI/LLM có thể hiểu đúng dữ liệu và sinh truy vấn chính xác theo yêu cầu người dùng (NL2SQL, NL2DAX,...).

Phần trách nhiệm phụ: tìm hiểu về Data Build Tool (dbt) để phục vụ công việc sắp tới. (Link Udemy để học:  The Complete dbt (Data Build Tool) Bootcamp: Zero to Hero)

* Data ready for AI nên có:

  * Good naming convention
  * Semantic model’s metadata

    * Business terminology
    * Hierarchies
    * Relationships (ontology)
    * Metrics \& Aggregation
    * Data lineage
* Relevant keyword 

  * Semantic / Knowledge Layer
  * Ontology-driven modelling
* Microsoft Fabric Data Agent

  * How it work
  * Semantic model best practice for Data Agent
  * Best practice for configuring a Data Agent.
* How to build semantic layer

  * BI Tools: modeling layer, define metrics, relationship
  * DBT: use metrics layer / semantic layer in dbt
  * Federated approach: implement knowledge graph/data virtualization tool that sít on top of your data sources.
